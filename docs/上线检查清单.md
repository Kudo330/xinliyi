# 上线检查清单

## 一、环境变量

- [ ] 已配置 `AI_PROVIDER`
- [ ] 如使用 OpenRouter，已配置 `OPENROUTER_API_KEY`
- [ ] 如使用 OpenRouter，已配置 `OPENROUTER_BASE_URL`
- [ ] 如使用 OpenRouter，已配置 `OPENROUTER_MODEL`
- [ ] 如使用 MiniMax，已配置 `MINIMAX_API_KEY`
- [ ] 如使用 MiniMax，已配置 `MINIMAX_BASE_URL`
- [ ] 如使用 MiniMax，已配置 `MINIMAX_MODEL`
- [ ] 已配置 `NEXT_PUBLIC_APP_URL`
- [ ] 如果启用微信，已配置 `WECHAT_TOKEN`
- [ ] 如果启用微信，已配置 `WECHAT_APP_ID`
- [ ] 如果启用微信，已配置 `WECHAT_APP_SECRET`
- [ ] 如果启用微信，已配置 `WECHAT_ENCODING_AES_KEY`

## 二、构建与质量

- [ ] 本地 `npm run build` 通过
- [ ] 本地 `npm run lint` 通过
- [ ] 生产部署日志无报错
- [ ] 首页 `/` 可正常打开
- [ ] 聊天页 `/chat` 可正常打开

## 三、AI 能力验证

- [ ] `/api/debug-ai-config` 返回的 provider/baseUrl/model 与预期一致
- [ ] `/api/chat` 在生产环境可返回正常内容
- [ ] 未配置密钥时会返回可读降级提示
- [ ] AI 超时或失败时前端不会白屏
- [ ] 危机场景文案已人工确认

## 四、微信接入验证

- [ ] 微信后台 URL 指向 `https://你的域名/api/wechat`
- [ ] 微信后台 Token 与 `WECHAT_TOKEN` 一致
- [ ] 微信校验 `GET` 请求已通过
- [ ] 微信文本消息 `POST` 请求已通过
- [ ] 关注欢迎语已确认
- [ ] 如使用明文模式，已确认当前配置方式符合后台设置

## 五、安全与运维

- [ ] 正式域名已启用 HTTPS
- [ ] 已关闭 `X-Powered-By`
- [ ] 已启用基础安全响应头
- [ ] 已确认服务日志不会打印敏感密钥
- [ ] 已准备异常监控或平台告警

## 六、合规与产品

- [ ] 页面中没有医疗诊断式承诺
- [ ] 已补充隐私政策页面
- [ ] 已补充服务条款页面
- [ ] 已确认紧急援助热线号码
- [ ] 已确认适用用户提示和免责声明

## 七、建议的发布后观察项

- [ ] 观察 AI 接口 24 小时失败率
- [ ] 观察微信回调是否有 403 或超时
- [ ] 观察首页到聊天页转化是否正常
- [ ] 观察高频输入场景下页面是否卡顿
