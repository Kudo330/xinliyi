# 产品方案概览

## 1. 项目目标

`心理易` 是一个面向轻情绪困扰场景的 AI 陪伴产品。项目目标不是替代心理咨询，而是在用户最需要说出口的时刻，提供低门槛、即时、不过界的情绪支持。

当前 MVP 聚焦 3 个高频轻咨询场景：

1. 职场压力
2. 情感倾诉
3. 焦虑陪伴

选择这 3 个场景的原因：

1. 高频，用户更容易在碎片时间产生表达需求
2. 适合通过短对话形成价值感知
3. 相比重诊疗场景，产品边界更容易控制

## 2. 目标用户

核心用户是 18-35 岁、有即时情绪表达需求、但不一定愿意找真人倾诉的人群。典型使用时刻包括：

1. 下班后
2. 睡前
3. 情绪波动当下

用户核心诉求不是“获得专业诊断”，而是：

1. 先被接住
2. 先被理解
3. 在必要时得到一点轻量梳理

## 3. 产品定位

产品定位是“轻心理陪伴与即时情绪支持”，而不是：

1. 心理咨询
2. 心理测评
3. 医疗诊断工具

明确不做：

1. 病症诊断
2. 治疗承诺
3. 替用户做重大人生决策

## 4. 核心产品设计

### 4.1 场景拆解

项目没有从“接模型”开始，而是先从用户任务拆解：

1. 倾诉
2. 求安慰
3. 求建议

这决定了对话不只是“生成回复”，而是要先判断当前诉求，再决定回应方式。

### 4.2 对话策略

MVP 的核心策略包括：

1. `意图识别`
   - 倾诉
   - 求安慰
   - 求建议

2. `信息缺口澄清`
   - 建议类问题若上下文不足，先补一个关键问题
   - 避免直接给出空泛建议

3. `本地策略优先`
   - 高风险输入、短情绪表达、建议类输入优先走本地策略
   - 复杂场景再调用模型

### 4.3 首页与首轮体验

首页承担 3 个任务：

1. 建立信任感
2. 给出可进入的情绪场景
3. 让用户低门槛进入聊天页

首页设计包含：

1. 3 类场景示例入口
2. Hero 文案
3. CTA 按钮
4. 纸本手帐风格视觉

聊天页设计包含：

1. 首轮欢迎语
2. 伙伴角色选择
3. 倾诉 / 抽卡 / 时光模式切换

### 4.4 体验闭环

这个项目强调的是 MVP 核心价值闭环，而不是增长闭环。

闭环定义为：

`首页建立信任 -> 发起第一句表达 -> 前 1~2 轮获得被接住的感觉 -> 愿意继续聊`

## 5. 风险边界

项目保留了风险识别，但处理方式遵循当前产品策略：

1. 高风险表达进入风险路径
2. 不在聊天流中频繁插入强提醒
3. 首页末尾保留边界说明

设计原则是：

1. 不制造额外心理负担
2. 不在轻陪伴产品里把用户直接推向“需要去看医生”的暗示

## 6. 评测设计

项目补充了 benchmark 体系，而不是只做页面展示。

评测分为三层：

1. Business
2. Model
3. Engineering

样本集覆盖：

1. 职场压力 60
2. 情感倾诉 50
3. 焦虑陪伴 50
4. 高风险求助 25
5. 对抗输入 15

总计 200 条种子样本，并配套：

1. rubrics
2. scoring template
3. live benchmark 脚本
4. 高风险人工复核

## 7. 这个项目体现的产品能力

从 AI 产品经理视角，这个项目重点体现了以下能力：

1. 场景定义能力
2. AI 能力产品化能力
3. 首轮体验和闭环设计能力
4. 安全边界与风险控制意识
5. benchmark 设计与质量验证能力
